메이븐을 이용해서 프로젝트의 폴더 구조를 생성하고, 필요한 의존 모듈을 관리했다.
메이븐은 프로젝트의 빌드와 라이프 사이클, 사이트 생성 등 프로젝트 전반을 위한 
관리 도구로서 많은 자바 프로젝트가 메이븐을 사용해서 프로젝트를 관리하고 있다.

메이븐 설치
메이븐을 이용해서 프로젝트를 생성하는 방법 방법과 폴더 구조, POM파일의 기본 구성, 메이븐 라이프 사이클 
그리고 메이븐 프로젝트를 이클립스에서 생성

메이븐 프로젝트 생성
groupid
프로젝트가 속하는 그룹의 식별값, 회사, 본부 또는 단체를 의미하는 값이 온다
페키지 형식으로 계층을 표현한다.
ex) com.bitcamp.app
artifactid
프로젝트 결과물의 식별 값, 프로젝트나 모듈의 의미하는 값이 온다.
ex) sample
version
결과물의 버전을 입력한다. 
ex) 1.0-SNAPSHOT
package
기본덕으로 생성할 패키지를 입력한다. 별도로 입력하지 않으면 groupid 와 같은 구조의 패키지를 생성한다.

src/main/java : .자바 소스코드 파일이 위치합니다.
src/main/resources : 프로퍼티나 XML 등 리소스 파일이 위치합니다. 클래스 패스에 포함됩니다.
src/main/webapp : 웹 어플리케이션 관련 파일이 위치합니다. (WEB-INF, JSP, HTML, CSS, JS 등)
src/test/java : 테스트 자바 소스 파일이 위치합니다.
src/test/resources : 테스트 과정에서 사용되는 리소스 파일이 위치합니다. 테스트 시에 사용되는 클래스 패스에 포함됩니다.

기본적으로 생성되지 않은 폴더라도 직접 생성해주면 됩니다.
예를 들어 src/main 폴더에 resources 폴더를 생성해주면 메이븐 리소스 폴더로 인식합니다.


- maven compile
- maven test
- maven package

#pom 파일 기본
메이븐 프로젝트의 루트 폴더에 위치한  pom.xml 파일은 프로젝트에 대한 정보를 담고 있는 파일입니다.

주요설정
프로젝트 정보 : 프로젝트의 이름, 개발자 목록, 라이선스등의 정보를 기술
빌드 설정 : 소스, 리소스, 라이프사이클 별 실행할 플러그인 등 빌드와 관련된 설정을 기술
빌드 환경 : 사용자의 환경마다 달라질 수 있는 프로파일 정보를 기술
pom 연관 정보 : 의존 프로젝트(모듈), 상위 프로젝트, 포함하고 있는 하위 모듈등을 기술
